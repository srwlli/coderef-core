## Step 3 — `rag-index` against fresh graph (DIVERGED)

**Command:** `CODEREF_LLM_PROVIDER=ollama CODEREF_RAG_LOCAL_ONLY=1 node dist/src/cli/rag-index.js --dir . --reset`

(`--reset` was required because the previous `coderef-vectors.json` carried 768-dim vectors from a prior Ollama run while the default provider — without `CODEREF_LLM_PROVIDER=ollama` — defaults to OpenAI 1536-dim. The CLI surfaces this as a clean `VectorStoreError: Dimension mismatch` and exits non-zero before touching the index, which is correct safety behavior.)

**Result:** ✅ exit code 0, status='success', 263/263 chunks indexed in 841s. **But:**

> ⚠ **DIVERGED:** AC-09 dynamic verification FAILS even though static verification PASSED at Phase 7 close. Static check on `graph.json` correctly counted 262 file-grain `header_missing` nodes; dynamic indexing-orchestrator path normalization fails to join those facets onto chunks at runtime, so all 263 files index without the AC-09 skip discipline firing.

### Phase 7 expected vs DISPATCH-003 actual

| Metric | Phase 7 closeout claim (static) | DISPATCH-003 actual (dynamic) | Divergence |
|---|---:|---:|---|
| `chunksIndexed` | ~1 (file with no header_missing facet) | **263** | +262 |
| `chunksSkipped` (with `header_status_missing`) | 262 expected | **0** | -262 |
| `IndexingResult.status` | `partial` expected | **`success`** | flipped |
| `validationGateRefused` | `false` | `false` (correct) | n/a |

### Root cause

`src/integration/rag/indexing-orchestrator.ts:519-545` (Path A facet enrichment) joins chunks to files via:
- **Chunk side:** `chunk.file` arrives as absolute Windows-backslash path (e.g. `C:\Users\willh\Desktop\CODEREF\coderef-core\index.ts`). Vector store IDs include this verbatim: `file:C:\Users\willh\Desktop\CODEREF\coderef-core\index.ts`.
- **Graph side:** `graph.json` `node.file` arrives as relative POSIX path (e.g. `index.ts`, `demo-all-modules.ts`).
- **Join attempt:** `normalizeChunkFile` strips `this.basePath` from the chunk side; with `rag-index --dir .` and `populate-coderef .`, the `basePath` and chunk-id formats fail to match (likely a combination of `path.resolve` producing absolute paths on the chunk side while basePath remains relative `.` or vice versa, plus backslash-vs-forward-slash normalization edge cases).
- **Result:** `facetByFile.get(key)` returns `undefined` for **every chunk**, so the worst-severity-wins aggregation never propagates `headerStatus='missing'` onto chunks. The skip rule at lines 562-565 (`if (chunk.headerStatus === 'missing' || …) skip with header_status_missing`) never fires.

### Static-vs-dynamic confirmation

Static check (run as a one-off node script reading `.coderef/graph.json` directly with no path normalization):
```
unique files with headerStatus='missing' in graph.json: 262
first 5: ['demo-all-modules.ts', 'examples/nextjs-api-route.ts', ...]
```
This **matches `validation-report.header_missing_count=262` exactly** — so the upstream pipeline is producing the right facet counts. The bug is isolated to indexing-orchestrator's chunk↔file path normalization at runtime.

### Disposition

- **No retroactive Phase 7 reopening.** Phase 7's closeout report explicitly stated the AC-09 verification was static-only because Ollama was down at close time. That caveat was load-bearing and accepted; the dynamic gap was always implicit. This is the verification gap the DISPATCH-003 smoke was designed to catch.
- **Stub filed for follow-up WO** (see § Stub for follow-up below).
- Hard rules upheld: ZERO src/ changes; ZERO doc changes. The fix is a follow-up WO, not a phase regression.
