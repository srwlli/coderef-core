## File Details

### index.json
- **Format:** JSON Array of `ElementData` objects
- **Count:** 1,270 code elements
- **Structure:** Each element contains `type`, `name`, `file`, `line`, `exported` properties
- **Purpose:** Flat index of all scanned code elements (functions, classes, methods, etc.)
- **Generated:** By `scanCurrentElements()` function in scanner module
- **Resource Sheet:** [`index.json-RESOURCE-SHEET.md`](./index.json-RESOURCE-SHEET.md) (252 lines)

### graph.json
- **Format:** JSON Object with `nodes`, `edges`, and `metadata` properties
- **Nodes:** Array of graph nodes representing code elements
- **Edges:** Array of dependency relationships (currently empty in this instance)
- **Metadata:** Scan metadata including languages, analyzer type, element count, timestamp
- **Purpose:** Dependency graph representation for relationship analysis
- **Generated:** By `buildDependencyGraph()` function in analyzer module
- **Resource Sheet:** [`graph.json-RESOURCE-SHEET.md`](./graph.json-RESOURCE-SHEET.md) (318 lines)

## Generation Workflow

1. **Scan Phase:** Scanner analyzes TypeScript/JavaScript files and generates `index.json`
2. **Analysis Phase:** Analyzer processes scanned elements and generates `graph.json`
3. **Storage:** Both files written to `.coderef/` directory in project root

## Usage

These files are consumed by:
- **Query Executor:** Reads graph.json for relationship queries
- **Context Generator:** Reads index.json for element discovery
- **Export Tools:** Reads both files for visualization and reporting
- **MCP Servers:** Provides codebase context to AI agents

## Maintenance

- **Auto-generated:** Files are regenerated on each scan/analysis run
- **Do not edit manually:** Changes will be overwritten
- **Version control:** Typically excluded from git (add to `.gitignore`)
- **Backup:** Consider backing up before major refactoring
